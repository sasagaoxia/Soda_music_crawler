# 修复说明 - 当前版本

## 本次修复点

### 1. 去掉 API 下载

旧版本说明里曾包含 API 策略，但你要求删除 API 下载，所以当前版本已移除 API 请求逻辑。

现在下载流程是：

```text
分享链接
→ 跳转真实页面
→ Selenium 无头打开页面
→ 解析页面内嵌 JSON
→ 搜索页面音频 URL
→ 下载
→ 检测时长
→ 转 MP3
→ 再次检测时长
```

### 2. 多线程剪贴板下载

旧版一首下载时容易阻塞下一首。现在剪贴板监听和下载任务分开：

```text
主线程：持续监听剪贴板
线程池：处理下载任务
```

默认线程数：

```python
MAX_DOWNLOAD_WORKERS = 2
```

### 3. 修复白色空白页面

Selenium 默认用无头 Chrome：

```python
HEADLESS = True
HIDE_BROWSER_WINDOW = True
```

并且 Windows 下隐藏 chromedriver 控制台窗口。

### 4. 60秒以内自动删除

新增：

```python
SHORT_AUDIO_THRESHOLD_SECONDS = 60
DELETE_SHORT_AUDIO = True
```

检测到短音频后会删除音频文件和同名 `.info.json`。

### 5. 增加一键启动脚本

新增：

```text
01_安装依赖.bat
02_启动剪贴板监听.bat
03_启动菜单.bat
04_手动连续下载.bat
```

减少命令行输入错误，特别是 `pip` 不在 PATH 的情况。

## 注意

本工具只适合处理你有权保存的内容。对于无版权、已下架、仅限 App 内播放或平台限制的内容，程序可能无法获取有效音频地址。
